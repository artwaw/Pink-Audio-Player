Detailed Description

Class managing the backend of the playlists.

Class internally uses QStandardItemModel to hold two separate sets of data: one for the "live view" playlist and other one, in form of QHash table, for the other playlists. I considered using in-memory Sqlite db, which would be faster and more memory efficient however refreshing the view would lead to problems with maintaining selection.

Internally class calls the wrapper around the TagLib to read the metadata from the files.

The class provides usuall getter methods to manage the content of each of the playlists.
